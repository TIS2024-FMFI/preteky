API

API - špecifikácia - v1.0 (7.4.2021)

Všeobecné informácie

  • API rozhranie je dostupné online na URL adrese https://is.orienteering.sk/api (ide o REST rozhranie)
    • K tejto URL adrese sa následne pridáva konkrétna metóda s jej parametrami (napr. /clubs/24), čiže výsledná URL adresa pre toto volanie bude: "https://is.orienteering.sk/api/clubs/24"
    • V niektorých metódach je možné poslať jeden, alebo viac filtrovacích parametrov (ako GET parametre) - ak je poslaných viac takýchto parametrov, tak sú aplikované spoločne cez logickú spojku A (AND)
  • Formát výmeny údajov je JSON, tj. pri GET requestoch API vracia JSON string, pri POST requestoch klient posiela JSON string (raw)
  • Podporované metódy: GET (pre získanie dát, napr. zoznam pretekov) a POST (pre ukladanie dát, napr. nová prihláška)
  • Prenos údajov je zabezpečený formou API kľúča - každý klub si môže vygenerovať 1x API kľuč na tejto stránke
    • Ak dôjde ku kompromitácii kľúča, alebo je potrebné ho pregenerovať, je to možné tak, že ho klub vymaže a následne znovu vygeneruje
    • Správu API kľúčov môže vykonávať iba osoba s právami "klubový administrátor"
    • Autentifikácia cez API kľúč prebieha vložením nového poľa do hlavičky každého HTTP requestu 
      • Pole s API kľúčom má mať názov „x-szos-api-key“ a hodnotu kľúča pre daný klub - viď napr.:
{
     'x-szos-api-key: DOHODNUTY_KLUC', //nové header pole
     'Content-Type: application/json', //existujúce header pole
}
  • Treba mať na zreteli, že všetky operácie sú vykonávané pod klubom, ktorého identifikuje daný API kľúč - teda nie je možné napr. cez API kľúč HBB vytvoriť klubovú prihlášku za klub EBB, a podobne
  • Stav spracovania požiadavky sa vracia HTTP Statusom, tj.:
    • 2xx - OK stav (v tomto prípade nemusí prísť žiadna textová odpoveď, samotný status 200 sa považuje za úspešnú operáciu),
    • 4xx - chybový stav: chybová hláška je v odpovedi v poli “message”:
      • Prístup zamietnutý: volanie vracia vždy: 403: Unauthorized
      • Resource not found: volanie vracia vždy: 404: Not found
      • Iná chyba (default): 4XX
    • 5xx - server error: chybová hláška je v odpovedi v poli “message”,
  • Kódovanie prenosu údajov cez API je UTF8
  • Ak nie je uvedené inak, textové polia majú maximálnu dĺžku 254 znakov
  • Na testovanie komunikácie zo strany klubov je možné použiť program POSTMAN (existuje aj ako rozšírenie do Google Chrome), na vyžiadanie poskytneme hotovú POSTMAN collection
  • Všetky operácie cez API sú zaznamenávané a vyhodnocované pre prípadné zneužitie
  • API v tejto fáze slúž hlavne na preberanie a čítanie informácií z IS SZOŠ a tiež na agendu prihlasovania na preteky
    • Momentálne nie je možné cez API spravovať preteky, prihlášky, alebo výsledky, avšak ak sa API osvedčí a o tieto funkcie bude záujem, bude možné ich dopracovať

Pokyny k používaniu

  • API kľúč a prístupy k API je zakázané šíriť tretím stranám - jediný oprávnený subjekt na využívanie API je klub, ktorý má daný API kľúč vygenerovaný
  • Dáta dostupné cez API IS SZOŠ patria SZOŠ a na ich šírenie tretím stranám je potrebný súhlas
  • API je možné využívať na nekomerčné účely pre potreby jednotlivých klubov (napr. zobrazenie kalendára, integrácia na agendu prihlasovania na preteky, ...)
  • Ak dôjde ku kompromitácii API kľúča, je potrebné bezpodmienečne vygenerovať nový kľúč
  • API prosíme využívať s rozumom a bez nadmerného zaťažovania - jedná sa o pilotnú prevádzku a optimalizácie na strane servera ešte budú prebiehať. Každopádne kluby prosíme o využitie cachovania, alebo inej formy dočasného ukladania dát z API na ich strane (pri údajoch, kde je to možné a prínosné)
  • Každú chybu, alebo návrh na zlepšenie prosíme nahlásiť na email: webmaster@orienteering.sk

Zoznam pretekov

Vráti zoznam pretekov podľa parametrov. Maximálny počet záznamov v odpovedi je 50, takže aj v prípade nastavenia parametra "count" na viac ako 50 bude počet obmedzený na 50 pretekov.

URL: /competitions
Metóda: GET
Vstupné GET parametre (filter):

  • „segment_id“ - ID segmentu, nepovinné, ak je zadané, volanie vráti iba preteky daného segmentu (viď číselník)
  • „classification_id“ - ID klasifikácie pretekov, nepovinné, ak je zadané, volanie vráti iba preteky danej klasifikácie (viď číselník)
  • „discipline_id“ - ID disciplíny, nepovinné, ak je zadané, volanie vráti iba preteky s danou disciplínou (viď číselník)
  • „club_id“ - ID klubu, nepovinné, ak je zadané, volanie vráti iba preteky, ktoré organizuje daný klub
  • „date_from“ - dátum, od ktorého sa majú preteky vyhľadávať (tvar yyyy-mm-dd), nepovinné (preteky sú zoradené od daného dátumu do budúcnosti)
  • „date_to“ - dátum, do ktorého sa majú preteky vyhľadávať (tvar yyyy-mm-dd), nepovinné (preteky sú zoradené od daného dátumu do minulosti)
  • „only_ranking“ - [0/1], nepovinné, ak je zadané "1", volanie vráti iba preteky, ktoré sa počítajú do rankingu
  • „count“ - počet záznamov, ktoré má volanie vrátiť

URL: /competitions/{COMPETITION_ID}/entries
Metóda: GET
Vstupné GET parametre (filter):

  • „club_id“ - ID klubu, nepovinné, ak je zadané, volanie vráti iba prihlášku daného klubu na dané preteky

Vytvorenie prihlášky pretekára na preteky

Uloží novú prihlášku pretekára na preteky. Prihlášku je možné iba pridať, nie upraviť. Ak je potrebná úprava, odporúčame prihlášku zmazať a pridať znovu. Prihláška je vytvorená pod klubom, ktorý vykonáva API a do pretekov podľa URL adresy. Prihlásenie pretekárov z iného klubu nie je možné. Na parametre a proces platia rovnaké pravidlá, ako pri prihlasovaní cez administráciu IS SZOŠ.

URL: /competitions/{COMPETITION_ID}/entries/save
Metóda: POST

Vstup:

{
    "registration_id": "0", //ID registrácie, alebo 0, ak prihlasujeme bez prepojenia na registráciu
    "first_name": "Medard", //meno
    "surname": "Féder", //priezvisko
    "reg_number": "PEZ0102", //registračné číslo
    "sportident": "123456", //sportident
    "comment": "prosím štart na konci", //poznámka
    "categories": [ //pole kategórií, kam sa prihlasuje (1-X, v poli sú iba tie kategórie, kam sa prihlasuje)
        {
            "competition_event_id": "312", //ID etapy pretekov
            "competition_category_id": "5339", //ID kategórie pretekov
        }
    ],
    "services": [ //pole služieb, ktoré si chce objednať (1-X, v poli sú iba tie služby, ktoré si objednáva)
